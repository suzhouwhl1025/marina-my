# Beta 测试问题清单

## 一、严重问题（阻塞性）

- **新装软件后无法被新终端识别**：安装新软件后，新建的终端检测不到该软件，必须重启整个 Marina 才能生效。
- **后端崩溃会拖垮所有终端**：一旦后端进程异常退出，所有终端会话全部丢失。需要调研后端的健壮性与崩溃恢复机制，确保后端不会轻易崩溃。
- **Ubuntu 支持缺失**：后续有一批"一生一芯"的小可爱们要评估本软件，Ubuntu 支持必须自己补上。动工前需先审计当前托盘模型如何迁移到 Ubuntu。环境预期是Ubuntu 22.04
- **多语言未实现**：至少需要支持中文和英文。

## 二、状态指示器

- 标签页下方标题栏中、标题前面的状态指示器经常不显示。
- 对 Vite 等长输出工具的状态判定不准确，需要在 Running 转暂停时引入 LLM 复核。
- 指示器应能区分"完成"和"失败"两种结束状态，依据进程返回值即可。
- 新建终端时指示器有抖动，需要 0.5 秒的消抖。
- 对于PS C:\Xilinx> Remove-Item "C:\Xilinx\2025_2" -Recurse -Force这种命令，没有输出，然后键盘又有消抖，最好是按回车或者一条命令开始执行的时候，自动让指示器指示正在工作中，然后用LLM复核保证不会切回来
- 黄灯感觉太让人焦虑了，可以考虑换个颜色

## 三、左侧边栏

- 已有"收藏""临时"分组，但缺少系统路径分组（如桌面、默认路径等常用位置）。
- "收藏""临时"等分组标题文字太浅、太小，比任何一个具体的终端标题还要小。
- 文件夹和终端的视觉区分度不够，展开结构不清晰，三角形图标看不清；在没有打开终端时尤其严重，几乎看不出层级。
- 当多个路径的末级文件夹同名时，没有做好区分。
- 侧边栏宽度无法拖动调整。
- 侧边栏顶部与右侧主区域顶部不对齐。
- 单击选中左侧文件夹后，点击其他地方无法取消选中状态。

## 四、界面与视觉

- 切换终端时不是直接定位到底部，而是会从上往下快速刷屏，明显不符合预期。

- 对于Claude code这种自绘终端的软件，不断闪动的字符后面经常出现闪烁的输入光标，这是非预期行为，而且这是一个真的输入光标闪动，因为他闪动的时候，输入法的UI位置也会同步闪动，不是一直都有这个问题，一般是运行一段时间之后才出现的，这是一个严重问题

- 标签页卡片建议加圆角以提升视觉效果。

- `window-x` 外层不需要包裹矩形。

- 标题栏底部不需要边框分割线。

- macOS 红绿灯按钮没有悬浮效果，建议做成可开关，而不是一直没有。

- macOS 风格下 Marina 和 window-x 的染色均不正确。

- 标签页斜体字，右上角会被遮挡

- 图形logo的设计存在视觉问题

  

## 五、工具栏与布局

- 需要在"完整页面"与"简易页面（仅显示终端本身）"之间切换的按钮；右键"在终端中打开"的入口也需要补上该选项。
- 标签页右侧需要一个终端工具栏（复制全部内容、刷新等），上述切换按钮可以放在这里。

## 六、模板功能

- 当前模板挂在系统层级，每个文件夹也需要专属模板，用于快速运行常用命令。
- 需要支持从终端历史快速反向推导出模板。
- 模板中输入命令后，需要支持 AI 一键补全。

## 七、主题

- 主题选择界面不要显示色卡。
- 加入 One Dark Pro 主题，再补充一些其他知名主题。
- Cutie 主题的配色非常糟糕，需要重新设计。
- 浅色主题对于claude code这种软件，甚至还有浅底白字的情况，这是不应该的，需要检查配色设计
- Claude code输入的内容完全没有适用主题，还是黑色底
- 右键菜单和弹窗没有正确适用主题

## 八、设置页

- 系统集成里那些"复制命令"按钮等调试性功能可以移除了。
- 设置中展示的数据目录与真实数据目录、文档描述不一致，需要对齐。
- 设置→数据页面设计不合理

## 九、其他

- 粘贴行为不够理想：粘贴到 Agent 时偶尔会提示"多行粘贴意外复制"。
- 新窗口打开下，右键打开终端的时候，新的窗口没有展开文件夹，选中终端标签页
- 路径存在性没有好好检查一下，没有考虑用户修改路径的情况
- win11右键菜单生效不稳定，需要提示用户重启
