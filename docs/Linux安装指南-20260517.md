# Marina Linux 安装指南(v0.1.0-beta.4 首发)

> 日期:2026-05-17
> 工单:BETA-003(a/b/c/d 全套)
> 产物:`.deb` 75 MB / `.rpm` 75 MB / `AppImage` 109 MB(均 x64)

## 三选一,挑你的发行版

| 你用什么 | 推荐格式 | 说明 |
|---|---|---|
| **Ubuntu / Debian / Linux Mint / Pop!_OS / Elementary** | `.deb` | 走系统包管理器,自动注册为默认终端,卸载干净 |
| **Fedora / RHEL / CentOS Stream / Rocky / openSUSE Tumbleweed** | `.rpm` | 同上,RHEL 系发行版的等价方案 |
| **其他发行版 / 不想动包管理器 / Live USB 试用** | `AppImage` | 单文件,`chmod +x` 后双击即跑,完全便携 |

产物路径(本机构建):

```
release/0.1.0-beta.4/Marina-0.1.0-beta.4-amd64.deb
release/0.1.0-beta.4/Marina-0.1.0-beta.4-x86_64.rpm
release/0.1.0-beta.4/Marina-0.1.0-beta.4-x86_64.AppImage
```

## 路径 A · Ubuntu / Debian(`.deb`,推荐)

### 系统要求

- Ubuntu 22.04 LTS 或更新(20.04 可能可以但未测)
- glibc ≥ 2.28(Electron 31 硬要求)
- x86_64 架构

### 安装

把 `.deb` 复制到目标 Linux 机器,然后:

```bash
sudo apt install ./Marina-0.1.0-beta.4-amd64.deb
```

`apt install ./local.deb` 比 `dpkg -i` 好:它会自动解决依赖,而 `dpkg` 会留一堆 `Errors were encountered`。

依赖会自动装(都是 Ubuntu 仓库内常驻库):

```
libgtk-3-0  libnotify4  libnss3  libxss1  libxtst6
xdg-utils   libatspi2.0-0  libdrm2  libgbm1
```

### 安装后会发生什么

`.deb` postinst 自动做三件事:

1. **创建 `/usr/bin/marina` 符号链接** → 指向 `/opt/Marina/marina`,让 `marina` 命令在 shell 可用
2. **注册为 x-terminal-emulator 候选**(优先级 50)。如果之前没显式 set 过,GNOME / GTK 程序"在终端中打开"开始用 Marina;手动切换:
   ```bash
   sudo update-alternatives --config x-terminal-emulator
   ```
3. **刷 desktop database**:`marina.desktop` 立刻被文件管理器和应用菜单识别

### 启动

- **应用菜单**:Activities → 搜索 "Marina" → 点图标
- **命令行**:`marina`
- **Nautilus / Nemo / Caja "在终端中打开"**:右键文件夹 → "在终端中打开"(若 Marina 是默认终端);Marina 启动后 cwd 自动定位到该文件夹

### 卸载

```bash
sudo apt remove marina-app
```

postrm 自动:
- 摘除 x-terminal-emulator alternative 候选
- 删 `/usr/bin/marina` 软链
- 刷 desktop database

## 路径 B · Fedora / RHEL(`.rpm`)

### 系统要求

- Fedora 38+ / RHEL 9 / CentOS Stream 9 / Rocky Linux 9 / openSUSE Tumbleweed
- glibc ≥ 2.28
- x86_64

### 安装

```bash
sudo dnf install ./Marina-0.1.0-beta.4-x86_64.rpm
```

或者(更老的发行版):

```bash
sudo yum localinstall ./Marina-0.1.0-beta.4-x86_64.rpm
```

依赖会自动装:

```
gtk3  libnotify  nss  libXScrnSaver  libXtst  xdg-utils
```

### 安装后行为

与 `.deb` 等价:`/usr/bin/marina` 软链 + `alternatives --install x-terminal-emulator` + desktop database 刷新。

### SELinux 提醒(Fedora / RHEL / Rocky)

这些发行版默认 SELinux enforcing。**Marina 首次启动可能被拦**(Chromium sandbox 命中 unconfined_t 转换)。**若启动失败:**

1. 看 audit log:
   ```bash
   sudo ausearch -m AVC -ts recent | grep -i marina
   ```
2. 若有 denied,临时切 permissive 验证:
   ```bash
   sudo setenforce 0   # 临时,重启失效
   marina              # 看是否能跑
   sudo setenforce 1   # 验证完切回
   ```
3. 若 permissive 下能跑,正路是写个本地 policy module。先开 issue,我们补 policy 文件到下一版 .rpm。

### 卸载

```bash
sudo dnf remove marina-app
```

## 路径 C · AppImage(单文件,所有发行版)

最干净的便携模式,不动系统目录,不需要 root,不会出现在应用菜单里。

### 安装

```bash
chmod +x Marina-0.1.0-beta.4-x86_64.AppImage
./Marina-0.1.0-beta.4-x86_64.AppImage
```

第一次跑会在 `~/.cache/Marina/` 解压一次,后续启动从 cache 走,快。

### 依赖

AppImage 几乎自包含,但需要主机有:

- `libfuse2`(运行时挂载 squashfs)。Ubuntu 22.04+ 默认有;Ubuntu 24.04 / Debian 12 需要手装:
  ```bash
  sudo apt install libfuse2
  ```
- glibc ≥ 2.28

如果跑 AppImage 报 `dlopen(): error loading libfuse.so.2`,装 `libfuse2`。

### 集成到应用菜单(可选)

AppImage 默认不进应用菜单。如果想要,装一个 AppImageLauncher:

```bash
sudo apt install appimagelauncher   # Ubuntu / Debian
sudo dnf install appimagelauncher   # Fedora
```

第一次跑 AppImage,AppImageLauncher 会问你"集成到系统?",选 Yes 即可。

### 卸载

直接删掉 AppImage 文件即可。清理 cache:

```bash
rm -rf ~/.cache/Marina
rm -rf ~/.config/Marina
```

## 设为系统默认终端

三种方式任选其一:

### 方式 1:Marina 设置内(BETA-003c)

启动 Marina → 设置 → 系统集成 → **"设为默认终端"** 按钮(仅 Linux 显示)

实际做了什么:

```bash
gsettings set org.gnome.desktop.default-applications.terminal exec marina
sudo update-alternatives --set x-terminal-emulator /usr/bin/marina
```

### 方式 2:GNOME 设置(图形)

设置 → 默认应用 → 终端 → 选 Marina(`.deb` / `.rpm` 装包后会出现在列表里;AppImage 需 AppImageLauncher 集成后才行)

### 方式 3:命令行(无图形依赖)

```bash
# GNOME / Cinnamon / MATE 系列
gsettings set org.gnome.desktop.default-applications.terminal exec marina

# Debian / Ubuntu(让所有 GTK 文件管理器找到 marina)
sudo update-alternatives --set x-terminal-emulator /usr/bin/marina

# Fedora / RHEL
sudo alternatives --set x-terminal-emulator /usr/bin/marina
```

恢复默认(切回 gnome-terminal 等):

```bash
gsettings reset org.gnome.desktop.default-applications.terminal exec
sudo update-alternatives --auto x-terminal-emulator   # Debian/Ubuntu
sudo alternatives --auto x-terminal-emulator          # Fedora/RHEL
```

## "在终端中打开"(Nautilus / Nemo / Caja)

设为默认终端后,在 Nautilus / Nemo / Caja 文件管理器里:

1. 右键任意文件夹空白处 → "**在终端中打开**"
2. Marina 启动,左侧路径栏自动选中该文件夹,右侧 PTY 的 cwd 也是该文件夹

工作原理:文件管理器通过 `gsettings ... terminal exec` 拿到 `marina`,然后 `fork() + chdir(dir) + execvp("marina", ...)`。Marina 主进程启动时检测 `process.cwd()`,如果不是根目录也不是 home,自动作为 first-session 的 cwd 加入(`src/main/index.ts` BETA-003c 实现)。

## 设置项位置

Marina 在 Linux 上把数据放在 XDG 标准位置:

| 内容 | 路径 |
|---|---|
| 配置文件(`settings.json` / `bookmarks.json` 等) | `~/.config/Marina/` |
| 日志(`marina.log`) | `~/.config/Marina/logs/` |
| 缓存 / 临时副本 | `~/.cache/Marina/` |
| 开机启动(若开启) | `~/.config/autostart/marina.desktop` |

主程序:`/opt/Marina/`(.deb / .rpm)或 AppImage 内置(便携模式)

## 常见问题

### Marina 启动 1 秒后闪退

最常见原因是 glibc 版本不够。`marina --version` 报 "GLIBC_2.28 not found" 之类的就是它。**升级发行版**(CentOS 7 / Ubuntu 18 老到不支持,需要 ≥ Ubuntu 22.04 / Fedora 38 / RHEL 9)。

### 找不到 marina 命令

- `.deb` / `.rpm`:postinst 没跑成功。检查 `ls -la /usr/bin/marina` 和 `ls /opt/Marina/marina`。若都缺,重装一遍。
- AppImage:AppImage 不会创建 `marina` 命令,你需要 `./Marina-*.AppImage` 直接跑,或者把 AppImage 复制到 `~/.local/bin/marina` 并 chmod。

### Nautilus 右键 "在终端中打开" 还是用 gnome-terminal

- 检查 `gsettings get org.gnome.desktop.default-applications.terminal exec` 是否是 `marina`
- 若上面输出对,但行为还是 gnome-terminal:GNOME 44+ 改走 xdg-terminal-exec spec,gsettings 不再生效。装 [xdg-terminal-exec](https://github.com/Vladimir-csp/xdg-terminal-exec) 或等 Marina 后续版本接入新 spec。

### Wayland 下输入法 / 焦点问题

Chromium / Electron 31 在 Wayland 下行为不完全等同 X11。已知:某些输入法切换可能丢焦点。临时绕过:

```bash
GDK_BACKEND=x11 marina
```

或在 Marina 启动器内固定:编辑 `~/.local/share/applications/marina.desktop` 把 `Exec=marina %U` 改成 `Exec=env GDK_BACKEND=x11 marina %U`。

### 关掉最后一个窗口,Marina 弹 modal "确定关闭?"

这是 BETA-003b 的行为(ADR-013):Linux 上无托盘,关掉最后窗口 = 退出。若仍有 alive session,弹 modal 二次确认避免误操作。**点取消窗口保留;点关闭 app 退出,session 全部终止。**

如果 modal 烦,**关掉所有 session 后再关窗口**就不会弹。

### 开机启动

设置 → 行为 → 开机启动 → 勾选。

实际做了什么:写 `~/.config/autostart/marina.desktop` 让 XDG 标准 autostart 拉起 marina。

## 验收清单(给用户自测)

按顺序走一遍,任何一步失败请开 issue 附该步骤的输出:

1. ✅ `.deb` / `.rpm` 装包成功,无 error
2. ✅ `which marina` 输出 `/usr/bin/marina`
3. ✅ `marina --version` 不闪退(可能没实现 --version 但不应崩)
4. ✅ 应用菜单能找到 Marina 图标
5. ✅ 启动 Marina,默认 shell 是 bash,能输入命令
6. ✅ `pwd` 正确显示用户 home(从应用菜单启动)
7. ✅ 关一个窗口(还有第二窗):安静关
8. ✅ 关最后一个窗口,有 alive session:弹 LastSessionConfirm modal
9. ✅ Modal 点取消:窗口保留;点确认:app 退出
10. ✅ 设置 → 行为 → 开机启动 打开 → `cat ~/.config/autostart/marina.desktop` 看到文件
11. ✅ Nautilus 右键文件夹 → "在终端中打开" → Marina 起,sidebar 自动定位到该文件夹

## 已知限制(v0.1.0-beta.4)

- **zsh / fish hook 不完整**:Marina 默认探测 bash 优先(优先级 0),zsh / fish 也能跑但 OSC 1337 cwd 跟踪暂未在 zsh/fish 下完整接通。修复路径在 docs/方案-BETA-003-Linux支持-20260517.md 标记的"后续 patch"中
- **macOS 仍未实现**:`getPlatformAdapter()` 在 darwin 抛 throw,需要后续 BETA 立项
- **SELinux policy 未预置**:Fedora / RHEL 用户首次启动若被 SELinux 拦,需手动 `setenforce 0` 或自行 audit2allow
- **Wayland-only 极端配置 / KDE Plasma 集成**:best-effort,不阻塞 release;BETA-003 方案 A 决策刻意克制
