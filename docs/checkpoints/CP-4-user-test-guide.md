# CP-4 用户测试指南

> 这份指南给开发者(也就是你)看。逐步操作,把任何不对劲的现象记下来反馈给 agent。
> 全部通过 → 回复 "CP-4 通过,可以开始 CP-5"。

## 准备

```powershell
git checkout checkpoint-4
npm install   # 若有新增依赖(本 CP 没新加)
npm test      # 应输出 209 个用例全部通过
npm run dev   # 启动应用
```

应用启动后:
- 至少能看到一个窗口
- 窗口里有侧栏(收藏 / 临时 / 最近 三栏)
- 主区有 + 按钮或终端(取决于上次状态)

> ⚠️ 后面所有"切到设置页"的步骤都是:**点侧栏底部的 ⚙ 设置 按钮**

## 测试 1:设置页骨架(预计 1 分钟)

1. 点齿轮 ⚙ → **整个 body 被设置页替换**(侧栏消失,左上角有 × 关闭按钮)
2. 左侧分类导航有 7 项:🎨 外观 / 🖥️ Shell 与启动 / 🚪 行为 / 💾 数据 / 🔗 系统集成 / 🔧 高级 / ℹ️ 关于
3. 点不同分类 → 右侧 panel 切换,标题对应
4. 点左上 × → 回到侧栏 + 主区视图

**通过条件**:7 个分类都能点开,× 能正常关闭。

**失败现象**:
- × 点了没反应
- 切分类时整个 body 闪一下白
- 侧栏没消失

## 测试 2:外观 - 主题(预计 2 分钟)

1. 进设置 → 外观
2. **主题** 那行显示 7 张卡片(Rose Pine / Dawn / Moon / Cutie / Business / Ubuntu / Windows Terminal)
3. 点其中任意一张 → **整个 UI + 终端颜色立即切换**(应该是肉眼可见的变化,不是延迟)
4. 切到 "Rose Pine Dawn"(浅色)→ 整个应用变浅色
5. 关设置回到主区 → **主题保持**(没回退)
6. 重启 `npm run dev` → 主题持久化(还是上次选的)

**通过条件**:7 套主题都能切,即时生效,持久化正常。

**额外检查 — Ubuntu / Windows Terminal 主题**:这两套是 CP-2 错误率较高的(色值表多),特别看一下 xterm 终端区的前景 / 背景 / 选中色是否合理(不要黑底黑字之类)。

## 测试 3:外观 - 跟随系统主题(预计 2 分钟)

1. 进设置 → 外观 → 勾选 "跟随系统主题"
2. 不要关设置页
3. 打开 Windows 设置(Win+I)→ 个性化 → 颜色 → 切换"模式"为"浅色"
4. 切回 EasyTerm → **应用应已切到 Rose Pine Dawn(浅色)**
5. Windows 设置切回"深色"→ EasyTerm 切回 Rose Pine(深色)
6. 取消勾选 "跟随系统主题" → 此时应用主题保持(不会再跟系统)
7. 手动选 Cutie → 这次系统切换深 / 浅色不再影响 EasyTerm

**通过条件**:勾选后系统切换深 / 浅色时 EasyTerm 自动跟;取消勾选后手动选生效。

## 测试 4:外观 - 字体 / 字号 / UI 缩放(预计 2 分钟)

1. 进设置 → 外观
2. **终端字体**:下拉里能看到 Cascadia Mono / JetBrains Mono / Consolas 等;装了的不带"(未装)",没装的带
3. 切到一个本机装了的字体(如 Cascadia Mono) → 终端区字体立即变
4. 选下拉最后一项 "— 自定义 —" → 出现一个文本输入框 → 输入 `'Comic Sans MS', monospace` 失焦后 → 终端字体变(Comic Sans 几乎所有 Win 都装,会真的变滑稽)
5. 切回下拉,重新选 Cascadia Mono
6. **终端字号**:8 - 24 数字框,改成 18 → 字号立即变大
7. **终端行高**:1.0 - 2.0,改成 1.5 → 行间距变大
8. **UI 字体**:类似终端字体,只影响 UI 区(侧栏 / 标签 / 按钮)。注:你回到主区才看得清效果
9. **UI 缩放**:75% - 150%,改成 125% → **整个 renderer 都会被 zoom**(包括终端)
   - 注意:这是 webFrame.setZoomFactor,Chromium 行为,会同时缩放终端字号 — 这是 V1 已知行为(见 self-test #3)

**通过条件**:每个字段改了立即生效,无延迟,无错位。

**失败现象**:改了没反应 / 数字越界但没回滚 / 自定义输入失焦后没生效。

## 测试 5:Shell 与启动 - 默认 shell(预计 1 分钟)

1. 进设置 → Shell 与启动
2. **默认 shell** 下拉应该列出本机检测到的 shell(至少有一个,通常 PowerShell 7 或 Windows PowerShell)
3. 切到不同 shell → 不会立即看到效果(默认 shell 在新建终端时才用)
4. 关设置 → 主区任意路径双击新建终端 → 应该用刚选的 shell 启动
5. 验证方法:在新建的终端跑 `$PSVersionTable`(PowerShell)或 `ver`(cmd)看版本

**通过条件**:下拉列出的路径与 `where pwsh` / `where powershell` 一致,选中后新建终端真用它。

## 测试 6:启动模板编辑(预计 5 分钟)

1. 进设置 → Shell 与启动 → 滚到 **启动模板** 区
2. 看到列表(初次 4 项内置:🐚 Shell / 🤖 Claude Code / ⚡ Codex / 📦 OpenCode)
3. 内置每项右边有 [设为默认] [编辑];自定义还有 [删除]
4. 点 Claude Code 的 [设为默认] → 该行多一个紫色"默认" tag
5. 点 [编辑] → 进入子页(标题"编辑模板:Claude Code",左上有"&lt; 返回")
6. 改 name 为 "My Claude" → 点保存 → 回到列表 → 名字已更新
7. 点 [+ 新建自定义模板] → 子页(标题"新建模板")
8. 填:name="echo test" / icon=🧪 / command="echo" / args="hi" / 启动方式选"先启动 shell" → 点创建
9. 列表多一行,自定义模板,有 [删除] 按钮
10. 关设置 → 在主区 + 按钮 / 模板按钮 应能看到"echo test"(会出现在选项里)
11. 双击随便一个收藏路径以新模板创建 session → 终端应执行 echo hi
12. 进设置 → 删除"echo test" → 列表减少一行
13. **试试删除内置 Shell** → 没有删除按钮(内置不可删,UI 不显示删除按钮)

**通过条件**:CRUD 全部工作,内置不可删,默认 tag 在切换时正确移动。

## 测试 7:行为分类(预计 3 分钟)

1. 进设置 → 行为
2. **启动时行为**:目前应是"打开一个窗口"(默认)
3. 切到"仅启动到托盘"
4. 关设置 → 关所有窗口 → 应用进入托盘模式 ✓ (CP-1 已验证)
5. 重启 `npm run dev` → **第一次启动时不会弹窗口**,只有托盘图标 → 单击托盘 → 弹窗
6. 切回"打开一个窗口"

7. **开机启动**:勾上 → 提示无 → 关闭应用 → 重启 Windows → EasyTerm 应自启动(可以观察 Win+R `shell:startup` 文件夹是空的,因为我们走 Run 注册表;用 `regedit` 看 `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 应有 EasyTerm 项)
8. 取消勾选 → Run 表里的项消失

> 注:测试 7 的开机启动验证需要重启 Windows,可以跳过。如不验,标"未验"。

9. **完全退出前确认**:勾上(默认就是)→ 创建至少 1 个 session → 右键托盘 → 完全退出 → **应弹对话框** "还有 X 个终端在运行,完全退出会关掉它们" → 点取消 → 应用不退
10. 再点完全退出 → 弹同样对话框 → 点"完全退出" → 真退
11. 重启应用 → 关闭所有终端(让它们 exited)→ 完全退出 → **不应弹对话框**(因为没有非 exited session)

12. **选中即复制**:进终端,鼠标拖动选中一段文字 → 粘贴到记事本应有内容 ✓
13. 进设置 → 行为 → 取消"选中即复制" → 回终端选中文字 → 粘贴记事本应是上次的内容(没复制新的)

14. **终端右键行为**:默认是"弹菜单"
15. 右键终端 → 弹菜单(复制 / 粘贴 / 清屏 / 搜索),复制在没选中文字时灰显
16. 切到"直接粘贴" → 右键终端 → 直接粘贴剪贴板内容(不弹菜单)

**通过条件**:每个开关改了真生效,无遗漏。

## 测试 8:终端搜索(预计 1 分钟)

1. 进任一终端 → 跑些命令产生输出(比如 `ls -la` 几次或 `Get-Process`)
2. 按 Ctrl+F → **终端右上角弹出搜索栏**(input + ↑ ↓ Aa ×)
3. 输入一个常见词(如 PowerShell 输出里的 "Process") → Enter → 高亮第一个匹配
4. Enter 继续 → 跳到下一个匹配
5. Shift+Enter → 跳到上一个
6. 点 Aa → 切换大小写敏感模式(再 Enter 应该匹配数变化)
7. Esc 关闭 → 焦点回到终端,可继续输入命令

**通过条件**:Ctrl+F 唤出 / Enter / Shift+Enter / Esc 全部工作,匹配高亮可见。

## 测试 9:数据 - 导出 / 导入(预计 3 分钟)

> ⚠️ 这一项**会重启应用并清空当前所有 session**,放最后做。

1. 先准备:在收藏里加几个路径,新建几个 session,创建一个自定义模板,改主题
2. 进设置 → 数据 → 点"导出…" → 弹保存对话框 → 选个位置(如桌面)→ 文件名默认 `easyterm-config-YYYYMMDD-HHMM.json`
3. 保存 → 设置页提示"已导出到 …"
4. 用记事本打开该 JSON → 应是合法 JSON,顶层有 `format: "easyterm-archive"` / `version: 1` / `settings` / `bookmarks` / `recent` / `templates`

5. 现在改设置页主题为不同的(如 Cutie),改某个设置项,删除导出时已有的某个收藏
6. 设置 → 数据 → 点"导入…" → 弹文件选择 → 选刚才导出的 JSON → 弹**二次确认** "导入将完全覆盖现有配置 ... 应用会重启"
7. 点"继续导入并重启" → **应用关闭并立即重启**
8. 应用回来后 → 主题应回到导出时的(Rose Pine 默认或你导出时选的)→ 收藏应回到导出时的状态 → 自定义模板还在

**通过条件**:导出文件可读;导入二次确认有效;导入后状态精确还原。

**失败现象**:JSON 损坏 / 导入没重启 / 重启后状态没恢复。

## 测试 12:打包产物 ⚠️ 已知问题(预计 5 分钟)

> 见 `docs/checkpoints/CP-4-self-test.md` "已知问题 #1":Windows 上跑 `npm run build` 会因 winCodeSign 工具的符号链接权限问题失败。

### 选项 A(推荐):启用开发者模式后构建

1. Win+I → 隐私和安全性 → 开发者选项 → 打开"开发者模式"
2. 关掉开发者模式确认对话框,**新开**一个 PowerShell(让权限生效)
3. 在项目根目录 `cd <repo>` 然后 `npm run build`
4. 等待 5 - 10 分钟(electron-builder 第一次会下 Electron 二进制 + native rebuild)
5. 期望产物:
   - `release/0.1.0-alpha.0/EasyTerm-Setup-0.1.0-alpha.0-x64.exe` — NSIS 安装包
   - `release/0.1.0-alpha.0/EasyTerm-Portable-0.1.0-alpha.0-x64.exe` — Portable 版
6. 双击 NSIS Setup → 完整安装向导(选位置 / 桌面快捷方式 / 开始菜单快捷方式)
7. 双击 Portable → 直接启动,无需安装

### 选项 B:管理员模式构建

1. PowerShell 图标右键 → 以管理员身份运行
2. 进项目目录 → `npm run build`
3. 同上

### 选项 C:清缓存后再试

1. 删除 `%LOCALAPPDATA%\electron-builder\Cache\winCodeSign\` 整个目录
2. 启用开发者模式或管理员
3. 重试

**通过条件**:`release/<ver>/` 下出现两个 .exe 文件,且双击各能启动应用。

**如果跑不通**:**不视为 CP-4 失败**,这是工程师机器 + Windows 默认权限问题,不是代码问题。在 self-test 里已记为 known issue。

## 测试 13:干净 Win11 上能装能跑(可选,预计 30 分钟)

5. 设置页能进 → 切个主题
6. 关闭 → 任务管理器看进程已退出

如果没干净 VM,标"未验",CP-5 时再补。

## 测试 14:回归 — 关闭单窗口仍不弹任何对话框(预计 30 秒)

1. 创建 1+ 个 session
2. 点窗口右上 × 关闭
3. **不应弹任何对话框**(即使有 session 在跑)
4. 任务管理器看 EasyTerm.exe 还在(纯托盘模式)
5. 点托盘 → 新窗口出来 → 之前的 session 在(无 owner,可点击接管)

**通过条件**:关闭单窗口绝无对话框。这是产品哲学红线。
