# Marina 键盘交互规范

> 本文档是 spec §7.2.2 的开发者实现锚。**任何 PR 改键盘行为必须先读完本文**。

**最后更新**:2026-05-24(KBD-1 全面整改)
**对应 spec**:`docs/软件定义书.md` §7.1 / §7.2.2 / §13.2

---

## 1. 哲学

Marina 是"鼠标优先"产品。键盘只承载**终端会话本身**的快捷键 — 不做 tab 切换、不做菜单加速、不做任何"应用内功能键"。所有 UI 操作走鼠标 / 右键菜单。

**理由**:目标用户是"愿意用鼠标的普通开发者",不是"愿意学快捷键的命令行专家"。坚持这条边界才能让产品自洽。

---

## 2. 键位清单(唯一权威)

代码侧数据源:[`src/shared/terminal-keybindings.ts`](../src/shared/terminal-keybindings.ts)
设置页直接渲染该数组,用户在"设置 → 行为 → 终端快捷键速查"卡片可见。

| 功能 | Windows / Linux | macOS | 备注 |
|------|-----------------|-------|------|
| 复制 | `Ctrl+C`(有选区时)/ `Ctrl+Shift+C` / `Ctrl+Insert` | `Cmd+C` / `Ctrl+Shift+C` | 无选区时 `Ctrl+C` 透传 SIGINT |
| 粘贴 | `Ctrl+V` / `Ctrl+Shift+V` / `Shift+Insert` | `Cmd+V` / `Ctrl+Shift+V` | 所有路径走同一个 `handlePaste` |
| 搜索 | `Ctrl+F` | `Cmd+F` | 唤出搜索栏,Enter=下一个,Shift+Enter=上一个 |
| 字号 | `Ctrl+Wheel` | `Cmd+Wheel` | 范围 8-24,本地立即生效,跨窗口 100ms debounce 广播 |
| 关搜索栏 | `Esc`(仅搜索栏可见时) | `Esc`(同) | 其他时刻 Esc 透传给 PTY(vim 等需要) |

**清单之外的所有键位必须透传 PTY**,renderer 层不得消费。

## 3. 实现不变式(必须遵守)

### 3.1 IME 守护
所有 keyboard 拦截在第一行必须有:
```ts
if (e.isComposing || e.keyCode === 229) return; // / return true(透传)
```

**理由**:中文 / 日文 / 韩文 IME composition 期间所有按键属于 IME 状态机,renderer 拦截会让 IME 卡死(用户报告"中文输入到一半敲空格 Enter 无反应"的根因)。微软拼音的标点 auto-convert 走 `keyCode 229 && !isComposing` 路径,所以两个信号都要查。

**适用范围**:
- `term.attachCustomKeyEventHandler` — 已守
- `Modal.tsx` window keydown — 已守(KBD-1)
- `ContextMenu.tsx` window keydown — 已守(KBD-1)
- 未来任何新加的 window keydown listener

### 3.2 唯一权威表
所有键位定义在 [`src/shared/terminal-keybindings.ts`](../src/shared/terminal-keybindings.ts) 的 `TERMINAL_KEYBINDINGS` 数组。

新增 / 修改 / 删除键位必须**同时**改:
1. spec §7.2.2 终端键位清单表
2. 本文档第 2 节
3. `TERMINAL_KEYBINDINGS` 数组
4. 单测 `src/shared/terminal-keybindings.test.ts` 的"清单覆盖 N 个键位"那条数字

任何 PR 漏改其中之一不予合并。

### 3.3 paste 路径单一入口
**唯一执行点**:`handlersRef.current.handlePaste()`

**触发来源**(全部走同一入口):
- `Ctrl+V` / `Ctrl+Shift+V` / `Shift+Insert` — 键盘 binding `consume-for-paste` action,return false 不发字节,浏览器 native paste 事件触发 capture-phase listener
- 语音输入程序(智模 / 闪电说等)— 模拟 Ctrl+V → 同上
- 浏览器右键菜单 → 系统 paste 事件 → capture-phase listener
- Marina 自绘右键菜单"粘贴" → 直接调 `handlePaste()`
- 剪贴板桥 → 直接调 `handlePaste()`

**绝不允许**:
- xterm 默认的 bubble-phase paste listener 触发(必须被 capture-phase `stopImmediatePropagation` 阻断)
- 在键盘 handler 内直接调 `handlePaste()` 然后让 native paste 也触发(双倍)

### 3.4 复制路径必清选区
所有复制路径完成后必须 `term.clearSelection()`,否则下次 Ctrl+C 因 `hasSelection===true` 走复制分支不发 SIGINT — 死循环 / 卡住的进程无法中断(CPB-C3 根因)。

代码侧:`copy-or-sigint` 和 `copy-and-clear` 两个 action 都已包含 clearSelection。新加复制类 action 必须保留此行为。

### 3.5 Overlay 栈
所有 modal 类 overlay(Modal / ContextMenu / 未来的对话框)必须:

1. mount 时调 `useOverlayRegistration(active)` push 进栈
2. window keydown listener 内先问 `isTop()`,只有栈顶才响应 Esc
3. unmount 时 useEffect cleanup 自动 pop

**理由**:Modal + ContextMenu 嵌套时 Esc 优先级由栈顶决定,行为可预测。否则靠 React 注册顺序决定谁吃 Esc,不可预测。

**例外**:终端搜索栏 Esc 走 `attachCustomKeyEventHandler` 路径(xterm 焦点路径),不接入 overlay 栈。理由:搜索栏 input focus 在终端 helper-textarea 上,与 Modal/ContextMenu 不冲突。

### 3.6 Replay 期间 focus 行为
session 切换时 scrollback replay 期间(典型 100-500ms),terminal-host 必须同时:
- `visibility: hidden` — 消除中间帧
- `inert` — 阻止 focus 落入子树

**产品决策**:replay 期间按键不响应是有意为之。符合"切换中"直觉,且避免 typeahead 误发到错位 focus(用户在 Sidebar 改名框残留焦点时切 session,敲 `ls` 三个字符可能进改名框)。

reveal 后 useEffect 主动 `helper-textarea.focus()` 让用户立刻可敲。

## 4. 反模式(不要做)

| 反模式 | 为什么不做 |
|---|---|
| 加 Ctrl+T 新 tab | spec §13.2 永远不做 — 鼠标优先 |
| 加 Ctrl+W 关 tab / 关窗口 | 同上 |
| 加 Ctrl+Tab / Ctrl+1..9 切 session | 同上 |
| 加 Alt+Letter 菜单加速键 | 同上;且 `Menu.setApplicationMenu(null)` 已禁默认菜单条 |
| 注册全局快捷键(globalShortcut.register) | spec §13.2 永远不做 |
| 在 `attachCustomKeyEventHandler` 直接写 if/else | 走 `TERMINAL_KEYBINDINGS` 表 |
| 在 modal / overlay 内不查 isTop 就响应 Esc | overlay 优先级会乱 |
| 在 window keydown 不查 isComposing 就响应 | IME 用户会被打断 |
| 复制后不 clearSelection | SIGINT 失效 bug(CPB-C3) |
| 在键盘 handler 内调 handlePaste | 跟 capture-phase listener 双倍 |
