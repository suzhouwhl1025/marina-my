## 1. 严重发现(P1 — 上线前强烈建议修)

### P1-1. xterm 快捷键闭包陈旧 → 设置变更不跟随

**位置**:`src/renderer/components/TerminalView.tsx:526-940`(mount effect)+ L592-653(`attachCustomKeyEventHandler`)

`attachCustomKeyEventHandler` 在 mount effect 里**一次性**注册,deps `[session.id, myWindowId]`。其闭包捕获 `handleCopy / handlePaste / handleClear / handleOpenSearch / handleCloseSearch` 这 5 个 `useCallback`,但它们的 deps 包含 **`bracketedPaste` / `modal` / `rightClickMode`** 等设置项,运行时会变。

**复现路径**:
1. 默认 `bracketedPaste=true` 启动一个 session
2. 进入设置页关闭 bracketedPaste,再回终端
3. **Ctrl+Shift+V** → 仍走旧 `handlePaste`(bracketedPaste=true 路径),仍用 `\x1b[200~..\x1b[201~` 包裹
4. **右键菜单 → 粘贴** → 走新 `handlePaste`(因 `handleContextMenu` 每次重渲都重建,L1073 deps 列表包含 handlePaste,菜单 onSelect 是最新引用)

**影响**:同一个"粘贴"动作,快捷键 vs 右键行为不一致,且与设置页脱节。

**修复**:同项目内已用过的 ref 镜像模式(L342 `toastRef.current = toast`)。例如:

```ts
const handlersRef = useRef({ handleCopy, handlePaste, handleClear,
                             handleOpenSearch, handleCloseSearch });
handlersRef.current = { handleCopy, handlePaste, handleClear,
                        handleOpenSearch, handleCloseSearch };
// attachCustomKeyEventHandler 内一律调 handlersRef.current.handlePaste()
```

### P1-2. EnvTextarea hover/focus 自动揭露 API key

**位置**:`src/renderer/components/SettingsView.tsx:1745-1777`(`EnvTextarea`)

模板编辑器中"环境变量"框默认 `filter: blur(4px)` 遮罩,但同时挂了 4 个自动取消遮罩的事件:

```jsx
onMouseEnter={() => setRevealed(true)}
onMouseLeave={() => setRevealed(false)}
onFocus={() => setRevealed(true)}
onBlur={() => setRevealed(false)}
```

**风险场景**:屏幕录制 / 直播 / 旁人路过 — 任何鼠标偶然 hover 都会立即明文显示 `ANTHROPIC_API_KEY=sk-...` 之类敏感值。设置项 hint 明确写"防被旁人看到 API key",但实现违背了这个承诺。

**修复**:移除 4 个自动 reveal 监听,只保留显式"显示/隐藏"按钮触发。

### P1-3. 模板删除用原生 `window.confirm()`

**位置**:`src/renderer/components/SettingsView.tsx:536`

```ts
if (!confirm('确认删除这个自定义模板?该操作不可撤销。')) return;
```

项目已统一走 `useModal().confirm`(TerminalView 大粘贴/ANSI/多行/拖文件元字符均用之,`Modal.tsx` 顶部明确写了"取代 window.confirm 是因为原生 dialog 关闭后焦点归还不可控 — CPB-P2 主诉根因")。这里漏改的原生 confirm:

- 关闭后焦点漂到 body,违背 FOC-5
- 不走主题样式,在浅色主题下尤其突兀
- 阻塞 renderer 渲染线程

**修复**:改用 `await modal.confirm({ title: '删除模板', message: ..., danger: true })`。

### P1-4. NumberInput 即改即被回流覆盖

**位置**:`src/renderer/components/SettingsView.tsx:1689-1692`

```ts
useEffect(() => {
  setText(formatPercent ? `${Math.round(value * 100)}` : `${value}`);
}, [value, formatPercent]);
// 注释承认:"行为略糙但 V1 可接受"
```

每当 main 广播 settings 变化(可能是别的窗口 Ctrl+滚轮调字号触发),本地正在输入的 text 被无条件覆盖。

**复现路径**:
1. 设置页 → 终端字号输入框,光标停在框内开始输入"15"
2. 另一窗口在终端里 Ctrl+滚轮(TerminalView L1182 trailing-debounce 广播 SETTINGS_UPDATE)
3. main 广播 SETTINGS_CHANGED → 此 effect 触发 → `setText` 把"15"刷掉

**修复**:
```ts
const inputRef = useRef<HTMLInputElement>(null);
useEffect(() => {
  if (document.activeElement === inputRef.current) return; // 用户正在编辑,不打扰
  setText(...);
}, [value, formatPercent]);
```

### P1-5. `setError` 被滥用为成功提示

**位置**:`src/renderer/components/SettingsView.tsx:1203-1213`(SystemIntegrationPanel.copyToClipboard)

```ts
setError(`已复制 ${label} 到剪贴板`);
setTimeout(() => setError(null), 1500);
```

把"成功"消息塞进 `errorMsg` 状态 — CSS 是红色 + `AlertTriangle` 图标,用户视觉上感知为"出错了"。同文件其他成功反馈走 `useToast().push({ kind: 'success', ... })`,本处不一致。

**修复**:
```ts
toast.push({ kind: 'success', message: `已复制 ${label} 到剪贴板` });
```

## 3. 架构 / 维护建议(非阻塞)

### A1. TerminalView 主 effect 接近 400 行

`TerminalView.tsx:526-940` 的 mount effect 集成了 scrollback 重放 / ResizeObserver / `attachCustomKeyEventHandler` / `SESSION_OUTPUT` listener / `onData` handler / Search disposable 等多个相对独立的功能。这些都强耦合 xterm 生命周期,拆分性价比不高 — 接受现状 + 完善注释已经做到位。建议在 P1-1 修复时引入 `handlersRef` 模式后,顺便把"长期不变的纯逻辑"(如 scrollback 重放、resize debounce)抽 hook。

### A2. SettingsView 1883 行单文件

7 个 Panel 内嵌单文件。已经按 Panel 分函数,完全可以拆 `components/settings/` 子目录 + 复用 `SettingRow / NumberInput / FontPicker / EnvTextarea / ExplorerIntegrationCard` 这套子组件。单文件下找控件不难,但开源后维护者上手会更友好。

### A3. global.css 2128 行单文件,无模块化

选择器无命名约定(`.tab-blank .tab-blank-icon` vs `.session-item.exited`),无 BEM/CSS modules。V1 可接受;长期建议:
- 按视图区域拆 `styles/{titlebar,sidebar,tabbar,terminal,settings,modal,toast}.css`,在 `global.css` 里 `@import`
- 或引入 CSS modules

### A4. 没有任何 renderer 单元测试

AGENTS.md 5.1 明确"前端不需要测",但 P1-1 / P1-4 这类纯逻辑 bug 单元测试就能发现。建议至少:
- store reducer 单测(已经是纯函数,易测)
- 关键 hook(NumberInput 同步、FontPicker 切换、handlePaste 各分支)单测

## 4. 亮点(值得保留的好做法)

1. **`store.tsx` 单一 reducer + Map 优化**:`sessions/state-changed` 只换变化的 session 引用,配合 `SessionItem` 的 `React.memo` 可精确跳过无关项重渲。注释把"抖动源 D"破法写得清楚。
2. **`useAppStateRef()`**:渲染不订阅、事件读最新的双层 Context 模式,是 SessionItem 性能优化的关键。
3. **prior bug fix 痕迹完整**:`CPB-*` / `FOC-*` / `RSZ-*` / `XTM-*` / `FLK-*` / `PER-*` / `TYP-*` 等 tag 把每条修复关联到具体审计条目和现象,便于回溯。
4. **`focus.ts` 跨组件焦点归还**:走 DOM 查询而非 ref 传递,避免组件边界耦合;rAF + 多重 skip 条件考虑周全。
5. **`Modal.tsx` + `ContextMenu.tsx` 显式 `previousActiveElement` 保存与归还**,且只在 `activeElement` 已落回 body / null 时归还(避免覆盖 onSelect 内主动改的焦点)— 经验感很强。
6. **CSS 变量 14 个语义槽** + 全部 `var(--x, #f0f)` fallback 配合 stylelint 检测缺失 var — 实用又克制。
7. **xterm scrollback 重放 + pending 协议** (`TerminalView.tsx:745-802`):mount 时先注册 output listener buffer 进 pending,invoke get-scrollback 拿到 lastSeq 后 replay,再把 pending 中 seq > lastSeq 的写入,切到直写模式。无去重原子写,设计正确。
8. **`encodeStringToBase64` / `decodeBase64ToBytes` 性能优化**(`TerminalView.tsx:1361-1390`):chunked apply 避开 V8 args 上限,Uint8Array.from + callback 触发 SIMD 加速 — 5MB 粘贴从 ~2s 降到 ~150ms。
9. **WebGL renderer + onContextLoss 回退**(`TerminalView.tsx:678-692`):GPU 抢占 / 驱动崩溃时自动回退 DOM renderer,不影响功能。
10. **bracketed paste 默认开 + ANSI 注入预检**(`TerminalView.tsx:416-432`):内嵌 ESC 字符走强警告 confirm,`sanitizePastedPreview` 把控制字符渲染成可见占位符避免预览本身欺骗用户。
