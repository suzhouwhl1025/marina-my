## 一、软件理念

**EasyTerm** 是一个面向 Windows 的、以路径为中心、对 AI agent 友好的桌面终端管理器。其核心论点是:

> "你的终端会话不应该因为关掉窗口就死掉。"

四条核心原则:

1. **Path 是稳定的,Session 是廉价的,UI 是临时的** —— 三栏分类(收藏/临时/最近)以路径为单位组织,session 跟随路径,窗口仅作"观察窗"。
2. **不让用户输入路径,只让用户点击路径** —— 文件夹选择器 + Explorer 拖入,杜绝 `cd` 打错。
3. **用户决策最少化** —— 路径在三分类间流转、session 在路径间归属、终端 resize、状态点显示,全部由状态机自动驱动。
4. **窗口与应用解耦** —— 关掉所有窗口仍以托盘守护进程形式运行,所有窗口平等,无"主窗口"。

目标用户:RTL/EDA 工程师、全栈开发者、并发运行多个 AI coding agent 的人 —— 他们的共同特征是"在多个不同路径下并发运行长时任务"。

## 二、功能摘要(已完成至 CP-4)

| 模块 | 已交付能力 |
|------|------------|
| **应用骨架** | Electron 31 多窗口 + 系统托盘 + 完全退出语义 |
| **PTY 后端** | `node-pty` + ConPTY,resize 反馈环修复,会话 owner 解耦 |
| **路径管理** | 三态状态机(收藏/临时/最近),容量与 LRU 淘汰,持久化 |
| **会话管理** | UUID + state(active/idle/exited),scrollback 保留,无时限"已退出"标签 |
| **路径与 cwd 解耦** | OSC 1337 增量解析器追踪 cwd 漂移,标签 ⚠️ 提示但不迁移 path(ADR-008) |
| **模板系统** | 4 内置模板 + 用户自定义,默认模板/启动参数/工作目录 |
| **设置系统** | 7 主题、字体枚举、Shell/行为/高级,深合并 + 校验,运行时 in-memory replace 导入(ADR-009) |
| **终端体验** | 多行粘贴确认、搜索栏命中数、右键菜单、tab 顺序、lucide 图标 |
| **Explorer 右键集成** | 文件夹 / 文件夹空白处右键 "在 Marina 终端中打开" — HKCU 注册表写入,新窗口 / 现窗口新 tab 可选(原计划 V1.2,提前到 v0.1.0-alpha.1)|
| **关于页 + 构建注入** | 版本号/git hash/构建时间 |
| **打包** | `electron-builder --win --x64` 通过验证 |
| **跨平台预留** | `platform/{windows,linux,macos}.ts` 适配层骨架 |

## 七、工程节奏小结
