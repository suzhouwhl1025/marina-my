# Explorer 右键集成 工作记录 — 2026-05-12

## 背景

软件定义书把 "Explorer 右键 → 在 Marina 终端中打开此文件夹" 划在 V1.2,设置页那个开关 V1 显示但 `disabled`。本日把它提前到内部上线版本 v0.1.0-alpha.1。

理由:Marina 是 path-centric 终端,从 Windows 自带文件管理器一键进入指定路径是核心入口体验之一,没它整个"以路径为单位"的工作流就少了最自然的起点。

## 决策记录

| 项 | 决定 | 备选 |
|---|---|---|
| 注册表 Surface | `HKCU\Software\Classes\Directory\shell\Marina` + `Background\shell\Marina` | 加 `Drive\shell\Marina`(右键 C:\ 等)— 不做 |
| 命令行参数 | `Marina.exe --open-here "<path>"`,Directory 用 `%1`,Background 用 `%V` | 用 `%V` 一统 — 实测在 Directory\shell 工作但不规范 |
| 行为可选 | `settings.systemIntegration.explorerOpenIn` = `new-window`(默认) / `recent-window-tab` | 写死新窗口 — 用户已运行 Marina 时强行新窗会感觉打扰 |
| 注册表 Icon 字段 | 不写 | 写 `"<exe>,0"` — 需要 build/icon.ico,本期独立任务 |
| 卸载行为 | 应用卸载不自动清 HKCU(用户级,无害);设置里关掉开关 = unregister | NSIS 卸载脚本里清 — V1 简化 |
| 菜单文案 | `在 Marina 终端中打开` | `在此处打开 Marina` / `Open in Marina` |
| 改名遗留 | 启动期静默 unregister `HKCU\...\EasyTerm` 两个 key | 不管 — 用户从 EasyTerm 升级会看到两条菜单 |

## 实现

### 1. WindowsAdapter(`src/main/platform/windows.ts`)

- 替换两个 `throw 'Not implemented (V1.2)'`,实装 `registerFileManagerIntegration(appExePath)` / `unregisterFileManagerIntegration()`。
- 调 `reg.exe` 走 `node:child_process.execFile`,数组参数,Node 帮处理 Windows quoting,避免 cmd.exe 注入。
- 加 `cleanupLegacyExplorerIntegration()` 删 EasyTerm 时代的两个 key,启动期静默调一次。
- 加 `runReg` helper + `__setRunRegImplForTest` 注入 mock,方便单测断言参数。

### 2. Settings(`src/shared/types.ts` + `src/main/settings-manager.ts`)

- `Settings.systemIntegration` 加 `explorerOpenIn: 'new-window' | 'recent-window-tab'`
- `DEFAULT_SETTINGS.systemIntegration.explorerOpenIn = 'new-window'`
- `validateSettings()` 加枚举校验

### 3. 主进程入口(`src/main/index.ts` + 新增 `src/main/argv-utils.ts`)

- `parseOpenHere(argv)` 提取到独立模块(`argv-utils.ts`),便于单测(import `main/index.ts` 会触发 `bootstrap()` 调 Electron API)
- `second-instance` handler 现在接收 `argv`,有 `--open-here <path>` 时走 `openPathInTerminal()`
- 冷启动:`whenReady` 末尾,`parseOpenHere(process.argv)` 命中时,在 `createWindowFromFactory()` 创建的窗口里 `webContents.once('did-finish-load')` 后再 `sessionManager.createSession`
- `settingsChanged` 监听 `systemIntegration.explorerContextMenu` 切换 → 即时 register/unregister
- 启动期:若 `settings.explorerContextMenu === true` 强制 re-register(用户重装/portable 改路径都让旧 command 失效)
- 启动期:`cleanupLegacyExplorerIntegration()` 静默清 EasyTerm 残留
- `sanitizeOpenHerePath(raw)`:trim、剥末尾反斜杠、`existsSync + isDirectory()` 校验,坏路径退回 `app.getPath('home')`

### 4. SettingsView(`src/renderer/components/SettingsView.tsx`)

- `SystemIntegrationPanel` 解 `disabled`,绑定 `settings.systemIntegration.explorerContextMenu`
- 加 "打开方式" radio group,`explorerContextMenu === false` 时禁用

### 5. 单元测试

- `src/main/platform/windows.test.ts` 8 个用例:register 调用序列、`%1`/`%V` 占位、unregister 容忍 not-found、register add 失败抛错、legacy cleanup
- `src/main/argv-utils.test.ts` 7 个用例:基本 happy path、缺尾巴、紧跟 flag、多次出现、空字符串、含空格
- `src/main/settings-manager.test.ts` 2 个新用例:`explorerOpenIn` 默认值 + 非法值校验
- `src/main/session-manager.test.ts` 1 处 mock settings 补齐 `explorerOpenIn` 字段

### 6. 版本号

`package.json` 0.1.0-alpha.0 → 0.1.0-alpha.1

## 手测脚本(打包后 .exe 上跑,不在 dev 模式)

1. **开启功能**:打开 Marina → 设置 → 系统集成 → 勾"启用"
   - 期望:勾选成功;打开 `regedit`,验证以下 4 个键值存在:
     ```
     HKCU\Software\Classes\Directory\shell\Marina         (default) = "在 Marina 终端中打开"
     HKCU\Software\Classes\Directory\shell\Marina\command (default) = "C:\Program Files\Marina\Marina.exe" --open-here "%1"
     HKCU\Software\Classes\Directory\Background\shell\Marina         (default) = "在 Marina 终端中打开"
     HKCU\Software\Classes\Directory\Background\shell\Marina\command (default) = "C:\Program Files\Marina\Marina.exe" --open-here "%V"
     ```

2. **右键文件夹本身**(直接点中某个文件夹图标右键):
   - 桌面新建 `测试-Marina` 文件夹 → 右键 → 看到 "在 Marina 终端中打开"
   - 点击 → Marina 当前窗口存在的情况下,按设置:
     - `new-window`:新开一个窗口,该窗口里有一个 PowerShell session,cwd = `测试-Marina` 路径
     - `recent-window-tab`:当前最近活动窗口里新开一个 tab,cwd 同上
   - 切换到另一种打开方式,重新右键验证

3. **右键文件夹空白处**:
   - 进入 `C:\Users\<you>\Documents` → 在空白处右键 → 同样的 "在 Marina 终端中打开"
   - 点击 → cwd 应该是 `Documents` 路径

4. **冷启动场景**:
   - 完全退出 Marina(托盘右键 → 完全退出)
   - Explorer 找一个文件夹 → 右键 → 在 Marina 终端中打开
   - 期望:Marina 启动 + 直接显示新窗口 + 该路径开了 session
   - 注:冷启动一律新窗口,即使 setting=`recent-window-tab` 也不适用(没有"最近")

5. **关闭功能**:设置 → 取消勾选
   - regedit 中两个根 key 应该消失(`Marina` 不再在 `Directory\shell\` 下,`Background\shell\` 下也没了)
   - Explorer 右键文件夹 → 看不到 "在 Marina 终端中打开"

6. **路径不存在的兜底**:
   - 直接在 PowerShell 跑 `& "C:\Program Files\Marina\Marina.exe" --open-here "Z:\nonexistent"`
   - 期望:Marina 启动,window 里 session 的 cwd 退回到 `%USERPROFILE%`(home),不闪退

7. **改名遗留清理**(仅对升级用户):
   - 提前用 regedit 手动写 `HKCU\Software\Classes\Directory\shell\EasyTerm`(default)="测试" + 一个 command 子 key
   - 启动 Marina
   - regedit 中 `EasyTerm` 两个 key 应消失(被 cleanup 删除)

## 已知不做(本期显式跳过)

- 不生成 `build/icon.ico`(独立任务) — 注册表 Icon 字段省略,Explorer 菜单图标走 exe 内嵌默认
- 不写 NSIS 卸载脚本清理 HKCU 注册表项 — 用户级,无害
- 不实现 `Drive\shell` 驱动器右键
- macOS / Linux 的 `register/unregisterFileManagerIntegration` 保持 stub
