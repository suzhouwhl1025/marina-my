## 一、明天首次验证(15 分钟)

### Step 1: 看菜单是否还在

电脑重启或注销重登录可能会冲掉 dllhost 缓存,但 MSIX 安装是持久的,菜单应该仍在新右键菜单里。

```powershell
# 在 PowerShell 7(pwsh)里跑,Windows PowerShell 5.1 看不到 Cert: drive
Get-AppxPackage Marina.ContextMenu | Format-List Name, Version, InstallLocation
Get-ItemProperty 'HKCU:\Software\Marina'
```

预期:看到 `Marina.ContextMenu_0.1.0.0_neutral__g27xq1j98qc22` + `InstallLocation = E:\projects\terminal\release\0.1.0-alpha.3\win-unpacked`。

### Step 2: 右键测试(主验证)

1. 打开 Explorer,导航到任意目录(随机选一个**中文路径** + 一个**纯英文路径**作对比)
2. 右键**文件夹本身**(不是空白处),看新菜单(圆角那个)是否直接显示「在 Marina 终端中打开」
3. **不要展开**「显示更多选项」—— 那是经典菜单,不算验证通过
4. 点击 → 应拉起 Marina,且左侧 sidebar 反映这个路径

如果只能在「显示更多选项」里看到,说明 MSIX 路径失败,新菜单未注册。按 README 的故障排查走。

### Step 3: 文件夹背景空白处右键

进入任意文件夹,在文件列表的空白处右键 → 同样应看到 Marina 入口。这覆盖 `Directory\Background` 那条 manifest 路径。

### Step 4: panic 防护快速测试

(可选,但**强烈建议**做一次)在 `command.rs` 的 `Invoke` 第一行临时加 `panic!("test")`,重新 build + install,然后点菜单 —— Explorer 不应崩。验证完删掉这一行。

## 三、UX 设计:用户开关功能时的引导

### 核心问题

证书信任 + MSIX sideload 是**有副作用的系统操作**:
- 自签证书要装到 `LocalMachine\TrustedPeople`(需要 UAC)
- 包装到 `WindowsApps`(写 C:\,UAC)

但用户在「设置 → 系统集成 → Explorer 右键集成」勾 checkbox 时,期望的是「一键开启」的体验,不是看到一堆 UAC 弹窗一头雾水。

### 设计原则

1. **零意外**:UAC 弹窗前必须有 Marina 内的明确说明,告诉用户接下来要点什么、为什么
2. **可拒绝且优雅降级**:用户拒绝证书安装 → 自动 fallback 到经典菜单方案,不让功能完全不可用
3. **一次性引导**:首次开启时弹引导对话框,之后开关只是注册/注销
4. **透明度**:告诉用户证书装到哪里(TrustedPeople 不是 Root)、为什么自签、什么时候会换成商业证书
5. **不偷做**:绝对不静默 `certutil -addstore Root` —— 杀软会报警,用户会不信任

### 决策流程

用户在 SettingsView 勾选「Explorer 右键集成」时:

```
检测 Windows 版本
  ├── Win10 / Win11 < 22000        → 走经典菜单(reg.exe HKCU)
  │                                    现有 registerFileManagerIntegration() 路径
  │
  └── Win11 22000+
      ├── MSIX 已安装 + 证书已信任    → 直接启用(无弹窗)
      │
      └── 首次开启 / 证书未信任       → 弹「首次安装引导」对话框 ↓
```

### 关闭功能时

```
用户取消勾选 → 弹一个轻量确认 toast/dialog:
  「将同时移除 Win11 新菜单和经典菜单的 Marina 入口,继续?」
  [ 确认 ] [ 取消 ]
  
确认后:
  - Remove-AppxPackage Marina.ContextMenu(新菜单)
  - reg delete HKCU\...\Directory\shell\Marina(经典)
  - 证书保留不动(下次再开不用重导)
```

### 失败处理

| 失败点 | 用户感知 | 处理 |
|---|---|---|
| 用户拒 UAC(证书导入) | 看到对话框关闭 | Marina 显示 toast:「证书未安装,已为你启用经典右键菜单」+ 走 fallback |
| 用户拒 UAC(MSIX 安装) | 同上 | 同上 |
| MSIX 部署失败(网络/磁盘) | toast + 错误码 | 提示「查看故障排查」 → 跳转 GitHub Wiki |
| Marina 升级后旧 MSIX 残留 | 启动期检查 | 类似现有 `cleanupLegacyExplorerIntegration` 的 silent 清理 |

### 设置页面状态展示

`SettingsView` 的 Explorer 右键集成区块在新逻辑下应展示:

```
☑ Explorer 右键集成
  
  ✓ Win11 新菜单    已启用    [重新安装]
  ✓ 经典右键菜单    已启用    [移除]
  
  证书信任状态: 已信任(2029-05-12 过期)
                                          [查看技术详情]
```

用 Win10 的用户只显示「经典右键菜单」那一行,不暴露 Win11 概念,避免认知负担。

### 4.4 资源打包

`extraResources` 在 `electron-builder.yml` 增加:

```yaml

  - from: src/shell-hooks
    to: shell-hooks
  - from: native/context-menu/build/MarinaContextMenu.msix  # 预构建产物
    to: context-menu/MarinaContextMenu.msix
  - from: native/context-menu/dev-cert.cer  # alpha 期自签证书
    to: context-menu/marina-cert.cer
```

公测版 OV/EV 证书到位后,`marina-cert.cer` 这一行可以去掉(商业 CA 证书早已被 Windows 信任,不需要单独导入)。

### 4.5 NSIS 集成

`build/installer.nsh`:

```nsi
!macro customInstall
  ; 默认不安装新菜单(让用户在 Marina 内主动开启),只写一个标记说"包文件已就位"
  ; 实际安装动作由 Marina 设置页面触发,通过 powershell 调 extraResources 里的脚本
!macroend

!macro customUnInstall
  ; 已有的 reg delete(经典菜单清理)
  DeleteRegKey HKCU "Software\Classes\Directory\shell\Marina"
  DeleteRegKey HKCU "Software\Classes\Directory\Background\shell\Marina"
  
  ; 新增:卸载 MSIX(无论用户当前是否开启,都清干净)
  ExecWait 'powershell -NoProfile -ExecutionPolicy Bypass -Command "Get-AppxPackage Marina.ContextMenu -ErrorAction SilentlyContinue | Remove-AppxPackage"'
!macroend
```

证书**不**在 NSIS 卸载时清,因为用户可能后续重装 Marina,留着省一次 UAC。
